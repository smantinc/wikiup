2. Context

Hello World at ${util.datetime.MM/dd/yyyy EEE HH:mm:ss}

For every specified requesting url, there are a set of contexts used to render its output.

Any variable with a '\$' token is meant to be replaced by its context value. Like the datetime below:

\${util.datetime} => ${util.datetime}

There are some predefined context for certain requests, and they are listed here

Here are more date examples below if you are really interested in it a lot.

Two and half hours before
\${util/datetime/HH:mm:ss,-2h30m} => ${util/datetime/HH:mm:ss,-2h30m}
Three days after 10-03-2008
\${util\datetime\MM/dd/yyyy EEE,3d,2008-10-03 00:00:00.0} => ${util\datetime\MM/dd/yyyy EEE,3d,2008-10-03 00:00:00.0}
